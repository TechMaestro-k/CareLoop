## Bugs found & fixed during simulation

1. **AMBER never created an escalation row** — `engagement.py` AMBER
   branch only sent a caregiver message; Insights API showed "0 amber
   alerts" forever. Fixed by inserting a `severity=amber, status=pending`
   row in the AMBER branch (T003).
2. **Payment status reverted to `pending` after `_mark_paid`** —
   `_mark_paid` saved `chosen_slot` with payment, then called
   `_ensure_handoff_summary(p)` which used the stale in-memory `p` and
   re-saved the old `chosen_slot` (no payment). Doctor accept then
   failed with 402 "patient has not paid". Fixed by mirroring the saved
   chosen_slot back onto `p` before the handoff summary call.
3. **Explicit refill requests were silently no-op'd** — a fresh patient
   with seeded supply still on file would never trigger the pharmacy
   agent. Added `force_refill` state flag wired through
   `trigger_refill_for_patient(force=True)` so an explicit patient
   request always produces an order.
